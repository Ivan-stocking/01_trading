# A股开盘30分钟强势股筛选程序

手动触发的 A 股短线选股工具，基于板块效应、趋势共振、量能确认与涨停基因等多维度规则，筛选具备 3-5 日持续上涨潜力的个股，运行后直接输出 3-5 只结果。

## 核心策略

- **持股周期**：3-5 天（短线）
- **触发方式**：手动运行（建议在交易日 10:00 开盘后 30 分钟执行）
- **输出形式**：控制台表格，返回前 3-5 只个股
- **交易哲学**：指数不可测，但资金抱团的板块永远不会缺席；个股必须背靠强势板块且具备涨停基因。

## 功能特性

- 板块生态分析：自动获取行业板块涨幅排名与梯队结构
- **双维度输出**：分别按「行业板块维度」和「概念板块维度」各输出 3-5 只个股
- 个股多级筛选：市值、趋势、乖离率、涨停基因、相对强度、长上影线等
- 分时量价确认：早盘量能爆发、均价线支撑、回踩缩量
- 集中式配置：所有阈值参数集中在 `config.py`，无需改代码即可调参
- 容错设计：备用数据接口、单股异常隔离、全链路日志
- **并发优化**：板块成分股、概念涨幅、个股筛选均使用线程池并发，配合 socket 超时与预检机制
- 轻量输出：运行后直接打印双维度表格，无需 HTML 页面

### 注意事项

- 建议在交易日 10:00 后运行，确保分时数据完整。
- 非交易日（周末、节假日）运行会自动跳过。
- **双维度输出**：板块维度优先输出有行业归属的股票；概念维度优先输出名称匹配热点概念关键词的股票。
- 每个维度通过数量不足 3 只时返回实际数量，超过 5 只取综合评分前 5。
- 降级模式下（东财板块成分股不可用），板块维度可能为空，仅概念维度有结果。

## 筛选规则

程序按以下顺序逐级筛选，任一条件不满足即淘汰：

### 一、板块生态（PlateAnalyzer）

1. 所属板块涨幅排名 ≤ `MAX_PLATE_RANK`（默认 30）
2. 板块梯队达标：涨幅≥5% 个股 ≥ 3 只，或涨停个股 ≥ 1 只
3. 个股涨幅 > 板块平均涨幅（板块内领涨地位）

### 二、个股基本面与趋势（StockFilter）

4. 排除 ST 股、停牌股、上市不足 60 日新股
5. 流通市值 ≥ `MIN_CIRCULATION_MKT_CAP`（默认 150 亿）
6. 日线 MA5 > MA10 > MA20，且三线向上倾斜（多头排列）
7. 周线 MA5 > MA10 > MA20（周线多头确认）
8. 乖离率在 `[-2%, 5%]` 范围内，或突破近 10 日新高（二选一）
9. 近 20 日有涨停记录（涨停基因）
10. 相对全 A 指数超额收益 ≥ `2%`
11. 近 3 日无长上影线且未突破的抛压信号

### 三、分时量价确认（MinuteDataProcessor）

12. 早盘（9:30-10:00）成交量 / 昨日全天成交量 ≥ `0.35`
13. 10:00 股价 ≥ 分时均价线
14. 回踩阶段缩量（回踩均量 < 上涨均量 × 0.8）
15. 9:45-10:00 期间最低价 ≥ 9:45 均价的 98%，且低于均价线时间不超过 20%

## 输出说明

通过全部条件的个股按 `ranking_score` 综合评分降序排列，取前 3-5 只输出。表格字段：

| 字段 | 说明 |
|------|------|
| 排名 | 综合评分排名 |
| 股票名称 / 代码 | 个股标识 |
| 涨幅 | 当日涨跌幅 |
| 流通市值 | 单位：亿 |
| 所属板块 | 同花顺二级行业 |
| 仓位建议 | 根据板块助攻情况给出 |

仓位建议规则：
- 板块龙头 + 突破前高 → **重仓**
- 板块涨幅≥5%个股 ≥ 5 只 → **正常仓位**
- 板块涨幅≥5%个股 ≥ 3 只 → **轻仓试错**
- 其他 → **跟风较弱，轻仓试错**

## 数据流

```
main.run_filter()
  ├─ PlateAnalyzer.run()           # 板块数据 → 排名 → 成分股映射 → 梯队分析 → 概念涨幅
  │    └─ fetch_plate_constituents() 内置预检：东财不可用时立即降级（节省 ~110s）
  ├─ get_all_a_index_change()      # 全A指数涨幅
  ├─ 并发遍历股票（ThreadPoolExecutor, STOCK_FILTER_MAX_WORKERS=10）:
  │    ├─ StockFilter.filter_stock()              # 日线/周线/基因
  │    └─ MinuteDataProcessor.check_all_minute_conditions(daily_df)  # 分时（复用日线）
  ├─ 按 ranking_score 降序排序
  └─ 双维度输出:
       ├─ 板块维度: 优先有行业归属的股票，取前 TOP_STOCK_MAX 只
       └─ 概念维度: 优先名称匹配热点概念的股票，取前 TOP_STOCK_MAX 只
```

## 注意事项

1. **交易日判断**：`is_trading_day()` 优先使用 akshare 交易日历接口（`tool_trade_date_hist_sina`），失败时降级为硬编码节假日列表（已覆盖 2024-2026 年，2026 年为预估）。
2. **数据源降级**：东财接口不可用时自动降级到新浪/同花顺接口。降级模式下部分功能受限（如板块成分股、流通市值），程序仍可运行但筛选精度降低。
3. **交易时段**：分时数据仅在交易时段有效，非交易时段运行结果不完整。`fetch_minute_data` 会自动按当日日期过滤，避免混入历史数据。
4. **流通市值单位**：akshare `stock_zh_a_spot_em` 返回的 `流通市值` 单位为**元**，程序内部自动除以 1e8 转为亿元后再与 `MIN_CIRCULATION_MKT_CAP` 比较。新浪降级模式下无此列，市值筛选自动跳过。
5. **日志查看**：运行日志写入 `stock_filter.log`。
6. **symbol 前缀**：新浪接口（分时/日线）要求股票代码带 `sh`/`sz`/`bj` 前缀，程序通过 `code_to_symbol()` 自动转换，用户无需关心。
