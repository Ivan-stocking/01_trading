# A股开盘30分钟强势股筛选程序

手动触发的 A 股短线选股工具，基于板块效应、趋势共振、量能确认与涨停基因等多维度规则，筛选具备 3-5 日持续上涨潜力的个股，运行后直接输出前 5 只结果。

## 核心策略

- **持股周期**：3-5 天（短线）
- **触发方式**：手动运行（建议在交易日 10:00 开盘后 30 分钟执行）
- **输出形式**：控制台表格，返回前 5 只个股（行业板块维度）
- **交易哲学**：指数不可测，但资金抱团的板块永远不会缺席；个股必须背靠强势板块且具备涨停基因。

## 功能特性

- 板块生态分析：自动获取申万二级行业涨幅排名与梯队结构
- **单维度输出**：仅按「行业板块维度」输出前 5 只个股（已移除概念维度）
- 个股多级筛选：市值、趋势、乖离率、涨停基因、相对强度、长上影线等
- 分时量价确认：早盘量能爆发、均价线支撑、回踩缩量
- 全量分析记录：所有扫描过的股票（含失败原因）追加写入 `stock_analysis_records.csv`
- 集中式配置：所有阈值参数集中在 `config.py`，无需改代码即可调参
- 容错设计：备用数据接口、单股异常隔离、全链路日志
- **并发优化**：板块成分股、个股筛选使用线程池并发，配合 socket 超时与预检机制
- 轻量输出：运行后直接打印行业板块维度表格，无需 HTML 页面

### 注意事项

- 建议在交易日 10:00 后运行，确保分时数据完整。
- 非交易日（周末、节假日）运行会自动跳过。
- **单维度输出**：仅输出行业板块维度，优先选择有板块归属的股票，取前 `TOP_STOCK_MAX` 只。
- 通过数量不足 5 只时返回实际数量，超过 5 只取综合评分前 5。
- 所有扫描过的股票（含失败原因）会追加写入 `stock_analysis_records.csv`。
- 降级模式下（板块成分股映射为空），遍历涨幅前 `DEGRADED_SCAN_COUNT`（100）只股票，跳过板块归属筛选。

## 筛选规则

程序按以下顺序逐级筛选，任一条件不满足即淘汰：

### 一、板块生态（PlateAnalyzer）

1. 所属板块涨幅排名 ≤ `MAX_PLATE_RANK`（默认 25，申万二级行业）
2. 个股涨幅 > 板块平均涨幅（板块内领涨地位）
3. 板块梯队结构（涨幅≥5% 家数、涨停家数）用于仓位建议生成，不作为淘汰门槛

### 二、个股基本面与趋势（StockFilter）

4. 排除 ST 股、停牌股、上市不足 60 日新股、非沪深主板（创业板/科创板/北交所）
5. 流通市值 ≥ `MIN_CIRCULATION_MKT_CAP`（默认 80 亿）
6. 日线 MA5 > MA10 > MA20，且三线向上倾斜（多头排列）
7. 周线 MA5 > MA10 > MA20（周线多头确认）
8. 乖离率必须在 `[-2%, 5%]` 范围内（必须条件）；突破近 10 日新高作为加分项记录，不作为门槛
9. 近 20 日有涨停记录（涨停基因）
10. 相对全 A 指数超额收益 ≥ `2%`
11. 近 3 日无长上影线且未突破的抛压信号

### 三、分时量价确认（MinuteDataProcessor）

12. 早盘（9:30-10:00）成交量 / 昨日全天成交量 ≥ `0.35`
13. 10:00 股价 ≥ 分时均价线
14. 回踩阶段缩量（回踩均量 < 上涨均量 × 0.8）
15. 9:45-10:00 期间最低价 ≥ 9:45 均价的 98%，且低于均价线时间不超过 20%

## 输出说明

通过全部条件的个股按 `ranking_score` 综合评分降序排列，取前 5 只输出。表格字段：

| 字段 | 说明 |
|------|------|
| 排名 | 综合评分排名 |
| 股票名称 / 代码 | 个股标识 |
| 涨幅 | 当日涨跌幅 |
| 流通市值 | 单位：亿 |
| 所属板块 | 申万二级行业 |
| 仓位建议 | 根据板块助攻情况给出 |

仓位建议规则（`generate_comment`，基于板块助攻强度）：
- 突破前高 + 板块涨停家数 ≥ `POSITION_PLATE_LIMIT_UP_HEAVY`(2) → **重仓**
- 板块涨幅≥5%个股 ≥ `POSITION_PLATE_ABOVE_5PCT_NORMAL`(5) → **正常仓位**
- 板块涨幅≥5%个股 ≥ `POSITION_PLATE_ABOVE_5PCT_LIGHT`(3) → **轻仓试错**
- 其他 → **轻仓试错**

> 注：`generate_comment` 仍会调用 `is_concept_active()` 调整仓位，但因概念分析步骤已移除，概念热度恒为"冷清"，"重仓"会被降级为"正常仓位"，备注固定附带"概念热点冷清(均值0.0%)"。

## 数据流

```
main.run_filter()
  ├─ PlateAnalyzer.run()           # 板块数据 → 全A股 → 排名 → 成分股映射 → 梯队分析
  │    └─ fetch_plate_constituents() 用申万成分股接口；映射为空时进入降级模式
  ├─ get_all_a_index_change()      # 全A指数涨幅
  ├─ 并发遍历股票（ThreadPoolExecutor, STOCK_FILTER_MAX_WORKERS=8）:
  │    ├─ StockFilter.filter_stock()              # 日线/周线/基因
  │    └─ MinuteDataProcessor.check_all_minute_conditions(daily_df)  # 分时（复用日线）
  │    └─ 同时收集 all_records 写入 stock_analysis_records.csv
  ├─ 按 ranking_score 降序排序
  ├─ _select_plate_stocks()        # 行业板块维度，取前 TOP_STOCK_MAX 只
  └─ print_results()               # 控制台打印行业板块维度表格（含前25板块列表）
```

## 注意事项

1. **交易日判断**：`is_trading_day()` 优先使用 akshare 交易日历接口（`tool_trade_date_hist_sina`），失败时降级为硬编码节假日列表（已覆盖 2024-2026 年，2026 年为预估）。
2. **数据源降级**：板块数据使用申万二级接口（`ak.index_realtime_sw`），成分股使用申万成分股接口（`ak.index_component_sw`），均不依赖东财。A股/日线/分时等接口在东财不可用时自动降级到新浪/同花顺。降级模式下（成分股映射为空）遍历涨幅前 `DEGRADED_SCAN_COUNT`(100) 只股票，跳过板块归属筛选。
3. **交易时段**：分时数据仅在交易时段有效，非交易时段运行结果不完整。`fetch_minute_data` 会自动按当日日期过滤，避免混入历史数据。
4. **流通市值单位**：akshare `stock_zh_a_spot_em` 返回的 `流通市值` 单位为**元**，程序内部自动除以 1e8 转为亿元后再与 `MIN_CIRCULATION_MKT_CAP` 比较。新浪降级模式（市值=0）下不淘汰股票（反推结果仅记录到 details，不作为淘汰条件）。
5. **日志与记录**：运行日志写入 `stock_filter.log`；全量分析记录（含失败原因）追加写入 `stock_analysis_records.csv`。
6. **symbol 前缀**：新浪接口（分时/日线）要求股票代码带 `sh`/`sz`/`bj` 前缀，程序通过 `code_to_symbol()` 自动转换，用户无需关心。
